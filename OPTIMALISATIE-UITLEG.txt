 ╔═══════════════════════════════════════════════════════════╗
 ║  SAMENVATTING                                             ║
 ╚═══════════════════════════════════════════════════════════╝

  Deze map bevat de geoptimaliseerde versie van je Lagenco-
  website. Alle originele bestanden zijn behouden en geen
  enkele functionaliteit is veranderd of verwijderd. Er is
  uitsluitend gewerkt aan de laadstrategie en renderinghints
  om de performance te verbeteren.

  Verwachte verbeteringen:
  • Snellere first paint (eerste zichtbare content)
  • Kortere time-to-interactive (pagina sneller bruikbaar)
  • Minder render-blocking resources in <head>
  • Sneller scrollen op lange pagina's (legal, reviews)
  • Eerdere netwerkverbindingen met externe diensten
  • Geen verandering aan visuele uitstraling of functies


 ╔═══════════════════════════════════════════════════════════╗
 ║  WAT IS ER AAN GEPAKT?                                    ║
 ╚═══════════════════════════════════════════════════════════╝

  ─────────────────────────────────────────────────────────
  1. HTML <head> OPNIEUW GEORDERD (alle pagina's)
  ─────────────────────────────────────────────────────────

  OORSPRONKELIJK (per pagina in <head>):
    <script src="https://cdn.tailwindcss.com"></script>
    <link rel="preconnect" href="https://fonts.googleapis.com">
    <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
    <link href="...googleapis.com/css2?family=Sora..." rel="stylesheet">
    <link rel="stylesheet" href="...font-awesome/6.4.0/css/all.min.css">
    <link rel="stylesheet" href="style.css">

  Probleem: ALLE vier de externe resources waren render-
  blocking. De browser wachtte met het schilderen van de
  pagina tot AL deze bestanden waren gedownload:

    • Tailwind CDN script      — ~100KB, JIT compiler die in
                                  de browser draait
    • Google Fonts CSS          — ~10KB, maar bevat @font-face
                                  verwijzingen naar 3 fonts
    • Font Awesome CSS          — ~70KB + webfont bestanden
    • style.css                 — ~73KB, kritieke CSS

  GEOPTIMALISEERD:
    <!-- 1. Preconnects (vroege netwerk-handshakes) -->
    <link rel="preconnect" href="https://fonts.googleapis.com">
    <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
    <link rel="preconnect" href="https://cdnjs.cloudflare.com" crossorigin>
    <link rel="dns-prefetch" href="https://images.unsplash.com">

    <!-- 2. Tailwind CDN (synchroon — nodig voor layout) -->
    <script src="https://cdn.tailwindcss.com"></script>

    <!-- 3. Kritieke CSS (synchroon — nodig voor first paint) -->
    <link rel="stylesheet" href="style.css">

    <!-- 4. Niet-kritieke CSS (asynchroon via print-media trick) -->
    <link rel="stylesheet" href="...googleapis.com/css2?family=Sora..."
          media="print" onload="this.media='all'">
    <link rel="stylesheet" href="...font-awesome/6.4.0/css/all.min.css"
          media="print" onload="this.media='all'">
    <noscript>
      <link rel="stylesheet" href="...googleapis.com/css2?family=Sora...">
      <link rel="stylesheet" href="...font-awesome/6.4.0/css/all.min.css">
    </noscript>

    <!-- 5. Hoofdscript (defer — haalt parallel op, draait na DOM parse) -->
    <script src="script.js" defer></script>

  WAAROM HELPT DIT:

  a) Preconnects & DNS-prefetch
     De browser kan nu direct bij het parsen van <head> de
     TCP/TLS-handshakes starten voor fonts.googleapis.com,
     fonts.gstatic.com, cdnjs.cloudflare.com en de DNS-oplossing
     voor images.unsplash.com. Dit scheelt typisch 100-300ms
     "cold connection" tijd bij de eerste request naar elke host.

  b) Async Font Awesome + Google Fonts (print-media trick)
     Het patroon media="print" onload="this.media='all'" zorgt
     ervoor dat de browser de CSS wel downloadt, maar de render
     NIET blokkeert. Zodra de CSS is geladen, wordt media naar
     'all' gezet en passen de stijlen toe.

       • Tekst wordt eerst in fallback fonts getoond
         (system-ui / -apple-system), dan direct geswap naar
         Sora / Inter / Fraunces zodra Google Fonts binnen is.
       • Iconen (<i class="fas fa-...">) zijn ~150-300ms niet
         zichtbaar tijdens het laden, daarna verschijnen ze.

     Dit is een volledig normale werking die webstandaard heet
     "progressive enhancement" en wordt door o.a. web.dev en
     Google Lighthouse aanbevolen.

  c) Tailwind CDN blijft synchroon
     Belangrijk: Tailwind is NIET async gemaakt. Reden: de
     HTML gebruikt Tailwind utility-classes zoals
     "hidden md:flex items-center gap-6" die de layout bepalen.
     Als Tailwind async zou laden, zou de pagina eerst verkeerd
     uitgelijnd verschijnen en dan plotseling verspringen
     (FOUC = Flash Of Unstyled Content). Dat is erger dan de
     winst die je zou behalen.

  d) style.css blijft synchroon, MAAR na Tailwind
     style.css is kritiek voor de eerste verf en MOET synchroon
     laden. De volgorde is bewust NA Tailwind, zodat style.css
     Tailwind utility-classes kan overschrijven (cascade). Dit
     was in het origineel ook zo en is zo behouden.

  e) script.js met defer i.p.v. einde-<body>
     Eerder stond <script src="script.js"> helemaal onderaan de
     <body>. Dat betekent dat de browser pas HALVERWEWE de
     HTML-parsing begon met het downloaden van script.js (99KB).

     Met <script defer> in de <head> start de download direct
     bij het parsen van <head>, in parallel met de rest. Het
     script draait pas NA de HTML-parse, vóór DOMContentLoaded.
     Dit scheelt typisch 200-500ms op trage verbindingen.

     De inline scripts onderaan index.html (admin tabs) en
     assortiment.html (search clear button) zijn behouden en
     werken nog steeds correct — ze draaien tijdens de HTML-
     parse, vóór het gedeferde script.js.


  ─────────────────────────────────────────────────────────
  2. STatische AFBEELDINGEN: LAZY LOADING
  ─────────────────────────────────────────────────────────

  Aan alle statische <img> tags in de HTML-bestanden is
  toegevoegd:
    loading="lazy"     — browser laadt de afbeelding pas als
                         deze near-viewport komt
    decoding="async"   — browser decodeert de afbeelding off-
                         main-thread, geen UI-stottering

  Specifiek toegevoegd aan:
    • index.html   — 2 categorie-thumbnails onderaan de pagina
                     (regel 360, 372) + pmMainImage in de
                     productmodal (regel 245, was verborgen)
    • assortiment.html — pmMainImage in productmodal (regel 205)
    • reviews.html — 9 review-foto's hadden dit al (correct
                     behouden door het script)

  Dynamisch gegenereerde afbeeldingen in script.js hadden
  loading="lazy" decoding="async" al — niets veranderd.


  ─────────────────────────────────────────────────────────
  3. CSS: CONTENT-VISIBILITY EN CONTAINMENT
  ─────────────────────────────────────────────────────────

  Aan het EINDE van style.css is een "PERFORMANCE OPTIMIZATIONS"
  blok toegevoegd. De originele CSS-regels zijn NIET veranderd.

  Toegevoegde regels:

  a) content-visibility: auto op lange, statische secties
     (.legal-page main, .legal-page .legal-toc, .reviews-grid,
      footer)

     Dit is een moderne CSS-property die de browser vertelt:
     "sla het layout-en-paint werk over voor dit element tot
     het daadwerkelijk in beeld komt". Voor pagina's zoals
     privacybeleid.html en retourbeleid.html (lange juridische
     tekst) scheelt dit aanzienlijk in initiële render-tijd.

     contain-intrinsic-size: auto 600px geeft de browser een
     size-hint zodat de scrollbalk niet verspringt als het
     element wordt gerenderd.

  b) contain: layout paint style op .product-card
     Isoleert elke productkaart: als er binnen een kaart iets
     verandert (bijv. hover-state), hoeft de browser niet de
     hele grid opnieuw te schilderen. Scheelt repaint-werk bij
     hover en scrollen.

     Belangrijk: box-shadow op de kaart ZELF wordt NIET afge-
     sneden (alleen descendants worden geclipt). De hover-
     schaduw werkt dus nog steeds zoals verwacht.

  c) will-change: transform, opacity op .modal-backdrop, .toast
     Promoot deze elementen naar een eigen GPU-laag, zodat de
     animaties (openen/sluiten van modal, in/out-slide van
     toasts) soepeler verlopen. De originele CSS had dit al
     voor .carousel-track en .reveal — deze zijn behouden.

  d) .page-enter * { transition: none !important; }
     Tijdens de 600ms page-enter animatie (body heeft class
     .page-enter) worden alle transitions uitgezet. Dit
     voorkomt "jank" tijdens de eerste paint. Na 600ms verwij-
     dert script.js de class en werken alle transitions weer
     normaal.

  e) @media (prefers-reduced-data: reduce)
     Nieuwe media-query: als de gebruiker "data saver" aan
     heeft staan (Chrome op Android, sommige andere browsers),
     worden niet-essentiële animaties uitgezet (.float-a,
     .float-b, .pulse-dot, .maintenance-anim, .progress-ring)
     en .reveal-elementen direct getoond. Bespaart data en
     batterij op mobiel.


  ─────────────────────────────────────────────────────────
  4. script.js: GEEN VERANDERINGEN
  ─────────────────────────────────────────────────────────

  Het script was al uitstekend geoptimaliseerd:
    • requestAnimationFrame voor alle scroll/resize handlers
    • IntersectionObserver voor scroll-reveal
    • passive event listeners waar mogelijk
    • throttling via "ticking" flag (60fps)
    • prefers-reduced-motion checks
    • GPU-acceleration via translate3d
    • cleanup van timers bij pagehide (bfcache-vriendelijk)
    • loading="lazy" decoding="async" op alle gegenereerde
      afbeeldingen

  Er is niets veranderd — dat was ook niet nodig.


 ╔═══════════════════════════════════════════════════════════╗
 ║  WAT IS ER NIET AAN GEPAKT (EN WAAROM)                    ║
╚═══════════════════════════════════════════════════════════╝

  De volgende dingen zijn bewust NIET veranderd:

  • Tailwind CDN blijft synchroon geladen
    Reden: async laden veroorzaakt layout-shift (FOUC), wat
    erger is dan de performance-winst.

  • Font Awesome blijft van de Cloudflare CDN gehaald
    Reden: lokaal hosten zou betekenen dat je de font-bestanden
    moet downloaden en toevoegen — dat verandert de bestands-
    structuur en is technisch een functionaliteitswijziging.

  • Google Fonts: 3 families met alle weights blijven laden
    Reden: weights weg-halen zou kunnen betekenen dat bepaalde
    teksten terugvallen op een andere weight = visuele
    verandering.

  • style.css en script.js zijn NIET geminified
    Reden: je wil de code kunnen blijven lezen en onderhouden.
    Minificatie scheelt wel bytes, maar het is onpraktisch voor
    een website die je zelf nog gaat bewerken.

  • Geen critical CSS inline in <head> gezet
    Reden: dat vereist per pagina een apart extract van
    kritieke regels. Het zou een ingrijpende wijziging zijn
    van de bestanden. De huidige aanpak (style.css synchroon,
    de rest asynchroon) levert al het grootste deel van de
    winst.

  • Geen service worker of caching-strategie toegevoegd
    Reden: dat vereist een apart bestand (sw.js) en HTTPS.
    Beter als een aparte stap te implementeren.

  • Geen afbeeldingen geoptimaliseerd (WebP/AVIF conversie)
    Reden: alle afbeeldingen komen van Unsplash CDN, die al
    optimalisatie levert. Lokale afbeeldingen (admin uploads)
    worden al via script.js geschaald en gecomprimeerd.

 ╔═══════════════════════════════════════════════════════════╗
 ║  ROLLBACK                                                 ║
 ╚═══════════════════════════════════════════════════════════╝

  Mocht je onverhoopt terug willen naar de originele versie:
  bewaar je originele zip-bestand. De geoptimaliseerde versie
  in deze map is een afzonderlijke set bestanden.

  Alle wijzigingen zijn backwards-compatible: er is geen
  functionaliteit veranderd, alleen laadvolgorde en rendering-
  hints. Als je een specifieke wijziging ongedaan wil maken,
  kun je per optimalisatie kiezen welke je behoudt:


  • Async Font Awesome + Google Fonts ongedaan maken:
    Verwijder in elke HTML in <head> de `media="print" onload=`
    en de `<noscript>` blokken, vervang door gewone
    `<link rel="stylesheet" href="...">` tags.

  • defer script.js ongedaan maken:
    Verwijder `<script src="script.js" defer></script>` uit
    <head> en zet `<script src="script.js"></script>` terug
    onderaan <body>.

  • CSS-performance-blok ongedaan maken:
    Verwijder alles NA het commentaar
    "PERFORMANCE OPTIMIZATIONS" aan het einde van style.css.

  • Lazy loading ongedaan maken:
    Verwijder `loading="lazy"` en `decoding="async"` attributen
    uit statische <img> tags in de HTML's.

 ╔═══════════════════════════════════════════════════════════╗
 ║  SUGGESTIES VOOR VERDERE OPTIMALISATIE (NIET UITGEVOERD) ║
 ╚═══════════════════════════════════════════════════════════╝

  Als je in de toekomst nog meer performance wil, overweeg
  dan de volgende stappen (deze zijn ingrijpender en daarom
  niet zonder overleg uitgevoerd):

  1. Tailwind lokaal compileren i.p.v. CDN
     - Installeer Tailwind CLI: npm install -D tailwindcss
     - Genereer een geminificeerde style-tailwind.css met
       alleen de utility-classes die je écht gebruikt
     - Voorkomt de JIT-runtime in de browser (~300ms winst)
     - Verwijdert de afhankelijkheid van cdn.tailwindcss.com

  2. CSS en JS minificatie
     - style.css: 73KB → ~25KB (gzip ~8KB)
     - script.js:  99KB → ~40KB (gzip ~13KB)
     - Gebruik esbuild, terser of een build-tool

  3. Font Awesome subsets
     - Je gebruikt ~30 iconen maar laadt de hele 70KB CSS +
       het complete webfont
     - Gebruik Font Awesome's "kit" met alleen de iconen die
       je gebruikt, of vervang door inline SVG's

  4. Service Worker voor offline caching
     - Cache style.css, script.js, Font Awesome lokaal
     - Snelle tweede-load (cache-first strategie)

  5. Afbeeldingen: betere formaten
     - Unsplash levert al WebP — overweeg AVIF voor nog
       kleinere bestanden
     - Voeg `width` en `height` attributen toe aan alle
       <img> tags om Cumulative Layout Shift (CLS) te
       voorkomen

  6. HTTP/2 Server Push of 103 Early Hints
     - Alleen relevant als je eigen hosting hebt
     - Laat de server style.css en script.js al pushen vóór
       het HTML-document volledig is verzonden
